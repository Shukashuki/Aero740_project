### Roy 的理解收穫（Phase 4 討論總結）
1. **Receding horizon 實際狀況**：每個 tick 規劃 1.6s（160 步），只執行 0.4s（40 步），然後重新規劃
2. **不同 DOF 的誤差差異**：同時計算但個別 DOF 的 tracking error 不同
3. **位置權重主導**：qpos 佔 cost >99.7%，qvel 幾乎可忽略
4. **計算負擔認知**：每 tick 最多 32 iterations × 1024 rollouts × 160 steps
5. **成功範例看不出 contact 影響**：需要跨 rollout/跨任務比較才能看出

### MPPI 概念討論
- **MPPI vs RRT**：表面類似（隨機採樣），但本質不同。RRT 做空間探索（長樹），MPPI 做 importance sampling（加權平均）。MPPI 近親是 CEM / particle filtering
- **部署時的 forward model**：MPPI 是 online planning，部署時需要即時 forward model（MuJoCo on GPU / learned dynamics / 簡化解析模型）。Receding horizon 幫助校正 model 誤差
- **Zero training, all inference**：MPPI 不訓練任何 weights，所有計算在推理時完成。換任務只需換 reference
- **初始 u 的重要性**：MPPI 做 local optimization，初始控制序列品質影響收斂。常見做法：PD controller / learned policy 粗估 → MPPI 精修

### Roy 總結的四個關鍵指標
1. opt_steps（優化收斂速度）
2. Reward 水準（追蹤品質）
3. Tick 邊界連續性
4. 跨 rollout 的 reward variance（穩定性）

## SPIDER Repo Study — Phase 5 進行中

### 已讀完的核心檔案

**`spider/optimizers/sampling.py`（MPPI 核心，真正的心臟）：**
- `sample_ctrls()` — 在 knot space 撒 noise → interp 到 horizon_steps → 加到 mean ctrl
- `make_rollout_fn()` — 構建 rollout 函數：save state → forward sim N×H steps → accumulate reward → restore state
  - terminate_resample：壞的 sample 被好的替換（copy simulation state）
- `_compute_weights_impl()` — 只取 top 10% samples 做 softmax，其餘權重為 0
- `make_optimize_once_fn()` — 單輪優化：sample → rollout（支持 domain randomization，取 min reward across DR params）→ compute weights → weighted average
- `make_optimize_fn()` — 多輪優化迴圈：beta_traj^i 退火 noise + early stopping

**`spider/simulators/mjwp.py`（MuJoCo Warp simulator 層）：**
- `setup_env()` — 建立 MJWPEnv（CPU model + GPU batched data + CUDA graph）
- `get_reward()` — cost function：weighted qpos tracking + qvel tracking + contact reward
- `_weight_diff_qpos()` — per-DOF weight（base_pos/rot/joint/object 分開設定）
- `get_terminate()` — 物體位置/旋轉誤差超過閾值就 terminate
- `step_env()` — apply perturbation → set ctrl → launch CUDA graph
- `save_state()`/`load_state()` — 用 data_wp_prev 做 state checkpoint（rollout 前存、後恢復）
- `sync_env()` — broadcast world 0 state 到所有 worlds
- `load_env_params()` — domain randomization：pair_margin, xy_offset, kp/kd

**`spider/io.py`（資料載入）：**
- `load_data()` — 讀 kinematic npz → interp 到 sim_dt → 尾巴 repeat horizon_steps+ctrl_steps 步

**`spider/interp.py`（插值）：**
- `interp()` — 用 F.interpolate 做 0th/1st/2nd order hold，把 knot points 插值到 horizon steps

**`examples/run_mjwp.py`（Entry point = `spider-mjwp`）：**
- Hydra config → load_data → setup_env → make_rollout/optimize_once/optimize
- 主迴圈：每 tick optimize → 執行 ctrl_steps → sync_env → receding horizon shift
- Contact guidance：compute_contact_point_delta → 修正 ctrl reference
- Gibbs sampling：bimanual 時先只 noise 右手、再只 noise 左手（交替優化）
- 輸出：trajectory_mjwp.npz + video
